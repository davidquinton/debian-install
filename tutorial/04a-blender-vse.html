Blender can look quite confusing for the first time, especially as it has so many functions in its main role of producing 3D animations. Its Sequence Editor has a slightly different approach to making movies than other editors and it's a very good idea to spend time with the video tutorials online first, so that things become obvious.

What's different

    Here are some main differences of Blender VSE to other non-linear video editors for Linux..

  • video clips are called strips, tracks are called channels, rendering is called animation!
  • video-audio is immediately split into "strips" when added, therefore moving a strip means that you must select both video + audio atthe same time, also for cutting at the same frame
  • there's no overlapping mode - when strips are overlapped the overlapping one highlights red, then on placing it it jumps to the end of the underlying strip - handy for placing strips exactly at end of others.

    • - means that vid/audio strips need to be cut at whatever frame should be first/last, then the gap between other strips closed up

    - the idea is that each strip should to be worked on by itself, not by another strip overlapping it deciding what its last frame should be.

  • selection is done by RMB, seek position is by LMB on the sequence editor.
  • the render does not assume that the total length of strips is the total length of the project to be made - rendering uses the frame range for length of output movie.



My Tips



Window arrangement

  • Open Blender and select Video Editing layout from first drop-down at top.
  • drag the preview from right over to the left and make the new far right window a Properties.
  • make the left window Graph if you like.
  • under File, Save Startup File - so that reopening Blender it will be ready for video editing.


Theme

  • open User Preferences in the left window
  • Themes tab
  • choose a preset - I selected Ubuntu and adjusted the graph and sequencer backgrounds a little lighter


Set Spacebar to play/pause animation..

  • open User Preferences in one window (select from type icon)
  • go to Input tab and expand Sequencer and Frame
  • click on Add New then expand it and paste "screen.animation_play" to the first box (which reads "none")
  • click on the second box twice and press Spacebar
  • make one for both Sequencer and Frame for the shortcut to be active when the mouse hovers these windows (including the preview window) its set to play!
    (this will stop Space from bringing up shortcut search!)
  • Click Save User Settings too.


Adding video strips

  • click on Add > Movie
  • select files (hold Shift for multiple)
  • once the strips are on the sequencer, check that the no. of frames for audio = no. frames for video - if audio shows more, go to the project properties and adjust the Frame rate to the correct value, e.g. 24, 25, 29.97 etc - so that the audio and video match up for frames.
  • if the video looks squashed or stretched, RMB on the video strip and go to Strip (below) > Render size to set the strip's dimensions to the properties window - as your saved properties may be set to 1080p resolution and the video you're working with is different.
  • if it still looks skewiff after that, go to the properties and adjust the Aspect Ratio to correct value (sometimes needs 1:1)



Zooming in on the Sequencer

  • Press Shift-B and drag a box with LMB to select an area to zoom in on
  • press Home to zoom out to see the whole length of strips
  • mouse scroll zooms in/out
  • Shift-MMB (middle button) + drag allows you to travel about along or up/down the Sequencer



Using the Timeline

  • if you are zoomed in quite close in the Sequencer, you can move to other places on it quite easily by clicking in the Timeline, below the Sequencer (you might need to pull it up to see it, and you may have to zoom out)
  • pressing Home while hovering the Timeline will zoom to the Frame range you have set, mouse scroll to zoom out/in
  • unfortunately, the Timeline doesn't show what positions the strips are placed on, or how much space they use, but it does show markers, so adding markers along the way would be useful



Markers

  • place a marker with M, RMB select one and X to remove it.
  • set hotkeys for Jump to previous/ Jump to next marker - RC on the menu item and select Add Shortcut, then type your shortcut, e.g Shift-, and Shift-. (in effect < and >)



Preview Range

  • Press P and drag an area to set the preview range - play will run through this area over and over, no stopping. That means you can keep playing a particular edit/effect, while you make adjustments to the effects to see the changes live.
  • Alt-P to clear the range.
  • to set the preview range more accurately, set the seek point to the start, then hover on the Timeline and press S , set the seek point to the end of the preview range and then hover the Timeline and press E



Selecting strips to move them about

  • RMB on the middle of a strip to select it, Shift-RMB to select others, or press B and drag a box with LMB to select all in the box (note that you should RMB without Shift on the first strip to cancel any previous selection and so as not add to the previous selected strips and later cause trouble)
  • you can press A to toggle select all
  • press G to grab them
  • they will now follow the mouse - LMB to place them, or RMB again to cancel the grab.
  • if you press the Shift key while moving the strip it will move much more minutely, in order to be more accurate.

you can't overlap strips, they will jump out to find space on the end.



Delete strips

  • RMB select strips and hit X or Delete, then confirm
  • Ctrl-Z will also undo the action



Cutting strips

  • seek to the cut point (Left/Right keys seek one frame)
  • RMB select the strip to be cut and press K for a soft cut (ends of strip can be extended again)
  • or Shift-K to make a hard cut (the ends will be gone)
  • select both video and audio strips to cut them together with K, or select them one by one and hit K for each one



Snapping strips


Shift-S snaps a selected strip to the seek point, or the end of a strip (stretches it out)

So, to move the start/end frame to the seek point (ie make a soft cut while removing the cut off part from the sequencer) ..
  • seek to the cut point
  • RMB on the end of the strip you want to shrink down
  • Shift-S to shrink the strip to the seek point


and if its the first strip in the sequence, and you remove some of the start frames, you will need to move to the new start frame with PgUp and use Frame > Set Start Frame (as there's no gap to remove)

if its a strip somewhere along the sequence the cut will make a gap - click LMB in the gap and use Backspace to close the gap.

If you press S (Slip Strip) you can then drag the content of the strip that you cut within its new frame length, in case you wanted to. (worth changing round S and Shift-S)

If you select the ends of a strip and press G to grab them you can extend them out, but if the strip then overlaps another strip and you place it with LMB, then the strip you extended will jump out to the end of the sequence to find enough space forewith that it might sit. It will not overlap or push out the strips it overlapped.

To get round that you need to insert gaps at the strip end with Shift-=, as much as you think you need to extend the strip ends.




Fade out/ fade in

  • Select the point to start the fade out to black.
  • in the properties to the right (press N) make sure Opacity is 1.0, RC and Insert Keyframe
  • move seek point to the end of the clip, click on the Opacity and set to 0, then RC and Insert Keyframe
  • Blender will make a smooth transition from one setting to the other between the keyframes, which you can see in the Graph (press Home to show full).
  • 0 opacity will be showing the base colour which should be black, unless other strips are under the strip on another channel.
  • on the graph window you can adjust the curve of the fade by dragging the line (with presets to choose from too)

You don't need to make a black colour strip and gamma cross effect (as the tutorials teach).


Transitions

You can use the above fade-out method pretty well to fade one strip and emerge another strip (the emerging strip should be placed under the fading strip) and this is more professional looking.

you can use an effect strip too..
  • set the strips you want to transition in an overlapping position on different channels
  • select both strips (video only or both video/audio)
  • go to Add > Effect strip > Gamma Cross
  • or you can use Wipe, which is more comical - and look in the properties to the right, scroll down to Transition and select which one you want

Colour correction

  • select a strips and go to properties on the right, scroll down to Add Modifier, select what you need and use the colour swabs below
  • to apply colour enhancements to multiple strips, without adding modifiers to each one, go to Add > Effect Strip >Adjustment Layer
  • then select the end of the strip, G to grab, drag to extend - or select and S to snap to the seek point
  • then you go to Add Modifier again in properties and adjust how you like



Multi-threading the render process

Man! Blender will not render on all threads together, it switches threads, using only 25% cpu power - thus renders take much longer than they should do and modern processing speed is not utilised.

Here's the answer (for now).. Pulverize

copy these scripts made by Hunter Perrin, from github.com/sciactive..

pulverize.php
pulverize_tools.py

(pulverize.py is also there but it doesn't show progress)

put them in your scripts dir or symlink to them in /usr/bin

then install php with,
 


For best results (concatenating with ffmpeg) set the video to MPEG/MP4/H264 (i.e. File format {container}/output file {extension}/encoding codec)

set the output file to a dir, without file name, and save to a blender project file.

cd projects/blender/
pulverize.php <blender_project_file> [<number_of_processes>] [<options>]

if you state too many threads, or none, it will default to half of real no. threads.

In order for the script to run properly with the portable app version, (ie not installed with package manager) you need to symlink the app in /usr/bin

ln -s path/to/your/blenderdir/blender /usr/bin/blender
